Soupault (soup-oh) is a static website generator.

It's a tool that takes a page template and a directory with page content files and builds a complete static website from it. This makes it easy to tweak the site layout without having to edit every page by hand. It can include external files in your pages, set the page title, create footnotes and tables of contents for your pagesâ€”in short, help you with tedious tasks that distract you from working on the site layout and content.

If you are already familiar with static generators like Jekyll or Hugo, you are likely wondering how soupault is different from them and why make another one. The key difference is that soupault doesn't use a template processor. Instead of generating HTML, it transforms and enhances it, using HTML element tree rewriting. Instead of making you supply page metadata in the “front matter”, it can extract metadata from your pages. It's more like an automated HTML editor that can emulate a human webmaster.

This allows new possibilities such as setting page title to the first <h1> of the page, no matter where it's located in the page, generating footnotes and ToC links with persistent anchors based on the id attribute of the heading or footnote paragraph elements, or inserting different scripts and styles into the <head> of certain pages.

Source and target elements for content extraction and insertion are identified using CSS selectors like h1, div#content, or .footnote. The way soupault works is similar to DOM manipulation in JS, except it doesn't need a browser and produces static pages that don't require any client-side scripts to work.

It also removes the artificial separation of theme and content and allows you to make every page look different if you feel like it. This makes soupault especially suited for the Web 1.0 revival. You decide how much is the theme and how much is the content, the minimal page template is just <html> <body> </body> </html>.

Despite its seemingly heavy-handed approach, soupault is reasonably fast and can build a site with a hundred pages in under a second.

What can you do with it?

  • Include files, HTML snippets, or output of external programs/scripts in your pages.
  • Automatically create tables of contents, footnotes, and breadcrumbs (fully static, no JS).
  • Delete HTML elements from certain pages.
  • Store pages in any format (Markdown, AsciiDoc...) that can be converted to HTML, if you specify a preprocessor for it.
  • Get page metadata in JSON and generate a section index/blog feed from it.
  • Add your own functionality with external scripts and plugins.

Download and install

Soupault is a single, self-contained executable (~6MB). The installation procedure is simple: just unpack the archive and you are ready to go.

Downloads:

Linux (x86-64, statically linked)
soupault-1.3.2-linux-x86_64.tar.gz
Microsoft Windows (7 and newer, 32-bit)
soupault-1.3.2-win32.zip
macOS
soupault-1.3.2-macos.tar.gz

Alternatively, you can download it from Github, along with GPG signatures.

Soupault is free software published under the MIT license. You can build it from source for any platform supported by the OCaml programming language and contribute to it. The source code is available from github.com/dmbaturin/soupault.

Why it's named soupault?

Soupault is named after the French dadaist and surrealist writer Philippe Soupault because it's based on the lambdasoup library. Its development is sponsored by the College of 'Pataphysics.Not really.

Graphics

The soupault logo is a stick horse. It's a reference to the meeting where Philippe Soupault et al. chose a name for their movement by opening a dictionary at a random word and landed on “dada (n.), a colloquial for a stick horse”, which is why they named it “dadaism”.

Who's behind it?

So far just me, Daniil Baturin, but everyone is welcome to send a patch/pull request or a suggestion. It has grown out of the bunch of ad hoc scripts that used to power my own website, and then I thought I can as well make it usable for everyone who finds other generators too annoying or too limiting.

If you find a bug or have a feature request, create an issue on Github. Alternatively, you can send an email to daniil+soupault@[my last name].org. Feel free to send me messages on Neocities as well.

Extensibility

Apart from built-in functionality, soupault offers a few ways to add your own.

Running external scripts

Soupault can run external scripts and include their output in the page. It can also pass arguments to scripts, so that you can run a script on the page source file (e.g. add automatically add last modification date to the page).

Index generators

While soupault, by itself, is not “blog-aware”, it can extract data from elements matching certain selectors from each page (e.g. title from a <h1 id="title"> and date from <time>) and feed it to an external script. This way you can make a blog, an Atom feed, or simply a list of all pages in a site section. You decide what data is extracted, from which elements, and what the script does with it.

Lua plugins

If you want new ways to modify the HTML tree of your pages, you can do it with Lua plugins. There are Lua functions for creating elements, deleting them, and setting attributes. The only downside is that the extension language is Lua 2.5 rather than modern 5.x.That's because soupault uses a research implementation named Lua-ML that includes many embedded interpreter design breakthroughs, but sadly isn't up to date with the official Lua spec.
