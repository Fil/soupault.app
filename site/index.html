Soupault (soup-oh) is a static website generator.

It's a tool that takes a page template and a directory with page content files and builds a complete static website from it. This makes it easy to tweak the site layout without having to edit every page by hand. It can include external files in your pages, set the page title, create footnotes and tables of contents for your pagesâ€”in short, help you with tedious tasks that distract you from working on the site layout and content.

Highlights

Soupault is different from existing projects in multiple ways:

  • It can work with existing HTML, no need for Markdown etc.
  • You can make all pages look different if you want.
  • A theme/template is just an empty HTML page.
  • It can preserve your existing site structure without breaking any links.

HTML first

Most website generators like Jekyll or Hugo are based on template processors. Soupault is different because it uses HTML element tree manipulation, similar to DOM manipulation in JS (getElementById etc.), so it's more like an automated HTML editor. Its “templates” are really just HTML pages with some empty tags, rather than HTML interspersed with template processor macros. Source and target elements for content extraction and insertion are identified using CSS selectors like h1, div#content, or .footnote. You can tell it to set the page title to the content of the first <h1> element, or insert templates/footer.html into <div id="footer">, and it will find those elements no matter where they are in the page.

Despite its seemingly heavy-handed approach, soupault is reasonably fast and can build a site with a hundred pages in under a second.

Perfect for Web 1.0

Many website generators are more like CMSes without a database and make it difficult to create free-form websites that people are turning back to Web 1.0 for. Soupault is intentionally designed to make it possible but still automate the tedious tasks.

You can make every page look different if you want. You can store your assets (images, CSS...) together with pages in site/ and they will be copied unchanged. You can create a site section simply by making a subdirectory and dropping pages into it. It's very easy to migrate to soupault from a handwritten website without breaking any internal or external links.

There is no artificial separation of theme and content. You decide how much is the theme and how much is the content, the minimal page template is just <html> <body> </body> </html>. You can setup a Web 2.0 style workflow with complete separation of the theme, content, and assets, but you don't have to.

Extensible

You can configure soupault to automatically run convertors to HTML from other formats (Markdown, AsciiDoc...) and include output of external scripts in your pages. You can also extend its functionality with LuaThe only pitfall is that the language is Lua 2.5, not modern Lua 5.x. That's because soupault uses a research implementation called Lua-ML that made some embedded intepreter design breakthroughs, but sadly isn't up to date with Lua standards. plugins.

You can also extract page metadata and export it to JSON for creating a list of all pages in a section or a blog feed. Elements to extract data from are configured using CSS selectors like everything else, so you can add an autogenerated index to an existing site without having to rewrite any pages.

Useful built-ins

There's built-in functionality for:

  • Including files or HTML snippets into any elements in your page.
  • Setting the page title to the content of any element from the page.
  • Creating tables of contents, footnotes, and breadcrumbs (all fully static, no JS required).
  • Deleting HTML elements from certain pages.

When you run soupault, it parses a page file, inserts its contents into an element in the template (by default, the <body> tag) and runs the page HTML through one or more “widgets”. Output of one widget can be used as input for another. You can order execution of widgets explicitly, e.g. to make sure ToC is created after all external files are included. It's also possible to set widgets to run only on certain pages or site sections (directories).

Download and install

Soupault is a single, self-contained executable (~6MB). The installation procedure is simple: just unpack the archive and you are ready to go.

Downloads:

macOS
soupault-1.4-macos.tar.gz

Soupault is free software published under the MIT license. You can build it from source for any platform supported by the OCaml programming language and contribute to it. The source code is available from github.com/dmbaturin/soupault.

Why it's named soupault?

Soupault is named after the French dadaist and surrealist writer Philippe Soupault because it's based on the lambdasoup library. Its development is sponsored by the College of 'Pataphysics.Not really.

Graphics

The soupault logo is a stick horse. It's a reference to the meeting where Philippe Soupault et al. chose a name for their movement by opening a dictionary at a random word and landed on “dada (n.), a colloquial for a stick horse”, which is why they named it “dadaism”.

If you are using soupault for your site or want to raise awareness of it, feel free to put a button there.

Who's behind it?

So far just me, Daniil Baturin, but everyone is welcome to send a patch/pull request or a suggestion.
