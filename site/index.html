Soupault (soup-oh) is a static website generator and HTML processor.

These are some of the things it can do for you:

  • Assemble a complete website from a template (an empty page) and a directory with page body files.
  • Insert files, HTML snippets, or output of external programs into your pages.
  • Create a table of contents from the page headings.
  • Create footnotes.
  • Automatically call convertors from other format (Markdown etc.) to HTML.
  • Delete unwanted elements.

How it's different from other tools? Website generators like Jekyll or Hugo generate HTML by filling the blanks in a template. Soupault parses HTML and then transforms and enhances it. It allows you to do things that are impossible with tools based on template processors. For example, you can insert the content of header.html before the first element in the page <body> and footer.html after the last element. Or insert the table of contents into a <div id="toc"> even if it's in a different place in every page. Or set the page title to the content of the first <h1> element. Instead of template tags like {{content}}, soupault uses CSS selectors to locate elements in the page, like body, .footnote, or div#content.

If you have a handwritten website, you can start using soupault right away to automate tedious tasks, and then gradually move the common layout to the template if you want to. You can easily mix unique and templated pages. Soupault can check if a file is a complete page or a page body (by checking if there's an <html> element in it), so it knows whether to insert a page in the template first or just apply HTML transformation rules to it.

If you are using another static site generator and have your content in Markdown/reStructuredText/whatever format, you can configure soupault to automatically convert your files to HTML before inserting it in the template.

While there's no built-in support for blogs, soupault can automatically extract page metadata and feed it to an external script, so you can create a blog, a simple list of pages in a section, or some custom index from your pages. See this post for a simple blog recipe.

You can also extend its capabilities with Lua plugins.The only pitfall is that the language is Lua 2.5, not modern Lua 5.x. That's because soupault uses a research implementation called Lua-ML that made some embedded intepreter design breakthroughs, but sadly isn't up to date with Lua standards. Plugins have full access to the element tree of the page, much like JavaScript, but without a browser.

Download and install

Soupault is a single, self-contained executable (~6MB). The installation procedure is simple: just unpack the archive and you are ready to go.

Downloads:

Linux (x86-64, statically linked)
soupault-1.5-linux-x86_64.tar.gz
Microsoft Windows (7 and newer, 32-bit)
soupault-1.5-win32.zip
macOS
soupault-1.5-macos.tar.gz

Soupault is free software published under the MIT license. You can build it from source for any platform supported by the OCaml programming language and contribute to it. The source code is available from github.com/dmbaturin/soupault.

Why it's named soupault?

Soupault is named after the French dadaist and surrealist writer Philippe Soupault because it's based on the lambdasoup library. Its development is sponsored by the College of 'Pataphysics.Not really.

Graphics

The soupault logo is a stick horse. It's a reference to the meeting where Philippe Soupault et al. chose a name for their movement by opening a dictionary at a random word and landed on “dada (n.), a colloquial for a stick horse”, which is why they named it “dadaism”.

If you are using soupault for your site or want to raise awareness of it, feel free to put a button there.

Who's behind it?

So far just me, Daniil Baturin, but everyone is welcome to send a patch/pull request or a suggestion. It has grown out of the bunch of ad hoc scripts that used to power my own website, and then I thought I can as well make it usable for everyone who finds other generators too annoying or too limiting.
